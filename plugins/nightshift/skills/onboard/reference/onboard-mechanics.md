# Onboard Mechanics — reference detail

Read this when the onboard SKILL.md step skeleton points you here. It holds the
extracted detail: the manifest-field table, the `.nightshift/` template tree, the
taxonomy clone + area-remap procedure, the full AskUserQuestion card set (with a
plain-prompt fallback), the completion-gate sentinel list, and the `.pack-meta.yml`
shape. Nothing here runs reviews — onboarding ends at a reviewed pack.

---

## A. The `.nightshift/` template tree

Copy from `${CLAUDE_PLUGIN_ROOT}/templates/.nightshift/`, then fill in for THIS repo.
Two lanes only — there is no `problems.yml`, no PM scaffolding.

```
.nightshift/
  manifest.yml              # from template — then fill in for this project
  .pack-meta.yml            # onboard writes this (see §E) — reconcile state
  .gitattributes            # onboard writes: metrics/**/*.jsonl merge=union
  registries/
    vectors.yml             # security — seeded this pass (clone+remap, §C)
    flows.yml               # design — only if Design lane selected (else omit)
  fixtures/
    personas.yml            # design — seeded from personas.example.yml if Design lane
  findings/
    suppressions.yml        # human-curated suppressions; starts as template
  metrics/                  # scaffolded empty (engine appends at run time)
    runs/                   # runs/<YYYY-MM>.jsonl   (append-only, one obj per run)
    findings/               # findings/<YYYY-MM>.jsonl (append-only)
    daily.jsonl             # append-only rollup; reader takes LAST line per (date,lane)
  dashboard.md              # DISPOSABLE projection — regenerated by runs
  trends.md                 # DISPOSABLE projection — CHANGELOG-style delta lines
```

`dashboard.md` and `trends.md` are disposable regenerated projections; do not
hand-author them. The durable truth is `metrics/runs/*.jsonl`, `metrics/daily.jsonl`,
`metrics/findings/*.jsonl`, and the git history of the registries.

### `.gitattributes`

Onboard writes `.nightshift/.gitattributes` containing exactly:

```
metrics/**/*.jsonl merge=union
```

This keeps concurrent appends conflict-free: branches just append; the reader dedupes
`daily.jsonl` on read by taking the last line per `(date, lane)`.

## B. Manifest fields (two-lane — NO PM)

Match every field name byte-exact to `${CLAUDE_PLUGIN_ROOT}/schemas/manifest.yml`.

| Field | Type | Onboard fills with |
|---|---|---|
| `pack_format` | int, REQUIRED | `1` (literal; gates future migrations). |
| `project` | string, REQUIRED | short pack slug (lowercase). |
| `repos[].path` | string, REQUIRED | repo path relative to `.nightshift` root. |
| `repos[].stack` | string, REQUIRED | detected stack tag (provenance: lockfile). |
| `stack_adapter.test` | string, REQUIRED | detected test cmd (confirmed at Card 2). |
| `stack_adapter.build` | string, REQUIRED | detected build cmd (confirmed at Card 2). |
| `stack_adapter.browser.tool` | string | Design lane only — e.g. `playwright-mcp`. |
| `stack_adapter.browser.base_url` | string | Design lane only — staging URL, never prod. |
| `allowlist` | list[string], REQUIRED | `Read, Grep, Glob`, the scoped test cmd, `WebFetch`; browser/MCP tool only if Design. |
| `linear.project` | string | Linear project key, or omit if "Skip / no Linear". |
| `linear.labels.security` | string | e.g. `sec-review`. |
| `linear.labels.ux` | string | e.g. `ux-flow` (Design lane). Label key stays `ux`. |
| `cadences.security` | keyword | `nightly` once vectors.yml is reviewed. |
| `cadences.design` | keyword | `weekly` if Design selected, else `off` (auto-deferred). |
| `window_budget_k.security` | int | default `6` (Card 3). Hard per-window ceiling K. |
| `window_budget_k.design` | int | default `4` (Design lane only). |

The lane key is `design` end-to-end (`cadences.design`, `window_budget_k.design`). The
agent file stays `ux-reviewer`, the Linear label key stays `ux`, registry `owner: design`.

### Three budget dials (point the user at these if they ask "what controls cost")

1. **K** = `window_budget_k[lane]` — a hard per-window ceiling. Never raised to clear
   backlog; overdue surplus routes to the digest/trend.
2. **per-agent `maxTurns`** + an in-prompt tool-call budget line.
3. **model tier per role** (agent frontmatter `model:`), globally overridable via the
   `CLAUDE_CODE_SUBAGENT_MODEL` env var (highest precedence — overrides frontmatter).

## C. Taxonomy clone + area-remap procedure

Seed a DRAFT `vectors.yml`; the human approves it at the REVIEW card before it's real.

1. Clone `${CLAUDE_PLUGIN_ROOT}/taxonomy/owasp-asvs.yml` into the draft
   `.nightshift/registries/vectors.yml`. This gives the generic ASVS spine (12 vectors:
   authn, session, access control, injection, secrets, deps, rate-limiting, crypto,
   logging, SSRF, file handling, security config) — each a registry entry conforming to
   `${CLAUDE_PLUGIN_ROOT}/schemas/registry-entry.yml` (`kind: vector`, `owner: security`,
   carries `asvs_ref`).
2. **Build the area→path map IN-MEMORY** from the detection preamble (globbed tree). The
   old `REPO_MAP.yml` reference is RESOLVED by dropping it — there is no `REPO_MAP.yml`
   file. For each base entry, replace its generic placeholder `area` globs with the real
   path globs for this repo. An entry with a wrong `area` is a silent blind spot.
3. **Extend with project-specific surfaces** the base taxonomy can't know. For each: set
   `id` (project-prefixed, e.g. `ND-SEC-05`), `title`, `kind: vector`, `area`, `weight`,
   `interval_days` (derive from weight: critical 7 / high 14 / medium 30 / low 90), and
   `owner: security`.
4. Leave `last_reviewed`, `status`, `linear` UNSET — those are engine-managed `(auto)`.

If the Design lane is selected, also seed a draft `flows.yml` (`kind: flow`,
`owner: design`) from the detected user-facing routes; otherwise omit the file.

## D. AskUserQuestion card set

`AskUserQuestion` runs in the TOP-LEVEL onboard skill only (unavailable to subagents).
Live limits: 1–4 questions per call, 2–4 options each, ≤12-char headers, `multiSelect`
supported, "Recommended" option first. Every option's DESCRIPTION carries provenance
(where the value came from). Render the "here's what I detected" summary BEFORE Card 1
and ask only deltas.

### Card 1 · Lanes (multiSelect)

Header `Lanes`. Options, Recommended first:

1. **"Accept all detected defaults and seed the pack"** [Recommended] — the fast path; a
   confident standard repo finishes in ONE interaction. Desc: "Security lane on, detected
   test/build/stack, K=6, no Linear unless detected. **Security-only fast path** — to also
   enable the Design lane, select both Security and Design options below instead of this
   option."
2. **Security** [pre-checked] — desc: "Security/assurance lane. Recommended default."
3. **Design** — desc: "Needs a staging env + seeded personas; seeds the design branch."

No PM option — the PM lane is dropped from core.

### Card 2 · Per detected repo (one card per repo; Recommended = detected value)

- **Test cmd** — Recommended = detected command + provenance (e.g. "from package.json
  scripts.test"). Options include "Other".
- **Build cmd** — Recommended = detected + provenance.
- **Stack** — Recommended = detected stack tag + provenance (e.g. "from go.mod").

### Card 3 · Filing & budget

- **Linear key** — a GENUINE question (cannot detect). Options: a typed key, plus
  "Skip / no Linear". Header `Linear`.
- **K (security)** — Recommended `6`. Desc: "Top-K entries reviewed per window. Hard
  ceiling — never raised to clear backlog." Header `Budget`.

### Design-lane branch (only if Design selected at Card 1)

Confirm/seed `fixtures/personas.example.yml` → `personas.yml`, and capture
`stack_adapter.browser.base_url` (staging). If Design is NOT selected, auto-defer with
ONE explanatory line (never a question): e.g. "Design lane deferred — set
`cadences.design: off`; re-run onboard once you have a staging URL + seeded personas."

### REVIEW card · proposed vectors (multiSelect)

Header `Vectors`. "Which proposed vectors look real?" — each draft vector an option;
unchecked vectors are dropped from the seeded `vectors.yml`.

**Reject a vector if**: its `area` glob maps to no real files in this repo / the attack
category doesn't apply to your stack / there's no owner available to triage findings from
it. Favor a short registry you'll maintain over a long one full of phantom coverage.

### FINAL card · Write pack

Header `Write`. Options: "Write pack" [Recommended], "Edit a field", "Other".

### Plain-prompt FALLBACK

If the runtime's AskUserQuestion limits differ from the above (1–4 Q / 2–4 options /
≤12-char headers), degrade to a plain numbered text prompt that asks the SAME content in
the same DETECT→CONFIRM→REVIEW→GATE→WRITE order: number each question, list 2–4 options
per question, keep option labels short, and put provenance in the option text. The flow
must remain identical; only the rendering changes.

## E. Completion gate — sentinel list

A DETERMINISTIC gate: grep the rendered pack for any surviving sentinel or
REQUIRED-empty key. Onboarding is NOT "done" until this is clean. Any survivor → emit a
precise BATCHED question (one card) naming each survivor and asking for the real value.

Sentinels to grep for:

- `my-project`
- `my-stack`
- `https://staging.my-project.example`
- `PROJ` (placeholder Linear key)
- unconfirmed `make test` / `make build` (the template defaults, when not chosen)
- any `# REQUIRED` marker left in a value position
- REQUIRED keys still empty: `project`, `repos[].path`, `repos[].stack`,
  `stack_adapter.test`, `stack_adapter.build`, `allowlist`, `pack_format`
- if Design selected: `personas.yml` still equal to the `.example` content, or
  `stack_adapter.browser.base_url` still the sentinel staging URL

## F. `.pack-meta.yml` shape + reconcile mode

Onboard writes `.nightshift/.pack-meta.yml` recording what was decided, so re-runs are
non-destructive:

```yaml
answers:            # the resolved interview answers (lanes, cmds, stack, K, Linear key)
engine_version:     # the nightshift plugin version that wrote this pack
template_ref:       # which template tree was cloned
pack_format:        1 # matches manifest.yml pack_format
```

### Reconcile mode (re-run with an existing `.nightshift/`)

1. Re-run the detection preamble.
2. Diff detected-vs-recorded (`.pack-meta.yml.answers` + the live tree).
3. Ask ONLY about drift — surfaces where detection now disagrees with what was recorded.
4. NEVER clobber `(auto)` engine-managed fields (`last_reviewed`, `status`, `linear`) or
   hand-tuned weights. Merge, don't overwrite.
5. Update `.pack-meta.yml` with the new answers + `engine_version`.
